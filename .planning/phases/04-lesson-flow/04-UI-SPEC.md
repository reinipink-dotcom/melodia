## Screen-by-Screen Contracts

### Screen 1: PreListenScreen

**Layout:**
```
SafeAreaView (midnight bg)
  StatusBar light-content
  NavBar row: [back chevron 36×36] [step indicator "1 of 6"] [spacer]
  ScrollView
    Hero card (LinearGradient #2A1A2E→#1E1430, borderRadius 16, padding 20)
      Row: [LevelBadge] [label "CONCEPT"]
      Module title (H2, 24px)
      Concept explanation (body 15px, mist color, 2–3 sentences)
    Section label "VOCABULARY TO LISTEN FOR"
    VocabList (vertical stack, gap sm)
      VocabRow × 3–5: [accent dot 6px] [Spanish 17px semibold accent-color] [English 13px mist]
  Fixed bottom CTA
    "I'm ready to listen" (coral gradient pill, full-width, 56px height)
```

**States:** Single state — no interactivity except CTA tap.

**Animations:** None on this screen. Keep it calm — user is preparing to focus.

---

### Screen 2: ListenScreen

**Layout:**
```
SafeAreaView (midnight bg)
  NavBar: [back chevron] ["Listening" label] [spacer]
  ScrollView
    Song card (surfaceContainer, borderRadius 16, padding 20)
      Album art placeholder (80×80, borderRadius 12, coral+22 bg, musical-notes icon)
      Song title (bodyMedium 16px)
      Artist name (caption 13px mist)
      "Listen on [Platform]" button (outline variant, platform icon left, full-width)
    Timer section (centered, lg vertical padding)
      Label "TIME LISTENING" (section label)
      Timer display (36px PlusJakartaSans 700, white) — MM:SS countdown
      Timer ring: circular progress (outer radius 72px, stroke 4px, surfaceHigh track, coral fill)
      "I'm listening" CTA → starts timer (coral primary, full-width)
      "I'm done listening" link-style text (mist color, 13px, appears after timer starts)
    Vocab reminder section
      Label "REMEMBER TO LISTEN FOR"
      Chip row (flexWrap, gap sm)
        VocabChip × 2–3: [surfaceHighest bg, borderRadius 20, px 10 py 6, Spanish word white 13px]
```

**Timer states:**
- **Pre-start:** Timer shows full duration (e.g. "3:24"), ring full coral. CTA = "I'm listening".
- **Running:** Timer counts down, ring depletes clockwise coral → track. "I'm done listening" appears below CTA.
- **Complete (auto):** Timer hits 0:00, haptic notification feedback, CTA changes to "Continue to Quiz →" (coral primary).

**"Listen on [Platform]" button:** Opens deep link via `expo-linking`. Platform icon from onboardingStore. Shows platform name: "Spotify" / "Apple Music" / "YouTube Music" / "YouTube".

**Back nav:** Confirm dialog "Exit lesson? Your progress won't be saved." → "Stay" / "Exit" options.

### Screen 3: QuizScreen

**Layout:**
```
SafeAreaView (midnight bg)
  NavBar: [close ✕ icon] [progress "Question N of 5"] [spacer]
  LinearProgress bar (full-width, 4px, coral fill — reflects question N/5)
  ScrollView
    Question card (surfaceContainer, borderRadius 16, padding 20)
      Question number label "Q1" (section label, mist)
      Question text (18px BeVietnamPro 600, white, lineHeight 26)
    Answer options (vertical stack, gap sm, mx md)
      QuizOption × 4 (uses existing component)
    "Check Answer" CTA (coral primary, disabled until option selected, full-width)
    — After answer checked —
    "Next Question" CTA replaces "Check Answer" (or "See Results" on Q5)
```

**Quiz flow states:**
1. **Idle** — 4 QuizOption in default state. "Check Answer" disabled (mist border, mist text).
2. **Selected** — Tapped option shows coral border. "Check Answer" activates (coral bg).
3. **Revealed** — Correct option: teal border + `${teal}20` bg. If wrong: selected option coral incorrect state + correct option reveals teal. All options disabled. "Next Question" appears.
4. **Transition** — "Next Question" tap: options reset to default, question slides in, progress bar advances.

**Back nav:** Same confirm dialog as ListenScreen — "Exit lesson?"

**No skipping:** User cannot advance without selecting and checking an answer.

**Haptics:** `expo-haptics` — `notificationAsync('success')` on correct, `notificationAsync('error')` on incorrect.

### Screen 4: QuizResultsScreen

**Layout:**
```
SafeAreaView (midnight bg)
  Scrollable content (centered, px lg)
    Spacer (xl)
    MascotIcon (size 120, centered, bounce animation playing)
    Spacer (lg)
    Reaction message (h2 24px, white, centered)
      "Perfecto!" → 5/5
      "Nice work!" → 4/5
      "Good effort!" → 3/5
      "Keep going!" → 0–2/5
    Spacer (md)
    Score display row (centered)
      Score number (48px ExtraBold, white) e.g. "4"
      " out of " (body 15px, mist)
      Total number (48px ExtraBold, mist) e.g. "5"
    Spacer (lg)
    XP card (surfaceContainer, borderRadius 16, padding lg, centered)
      Row: [star icon amber 20px] ["XP Earned" label mist] [XP number 28px amber bold]
    Spacer (xl)
  Fixed bottom CTA
    "Continue to Reading →" (coral primary pill, full-width, 56px)
```

**MascotIcon behavior:** Always showing bounce animation. Size 120px on this screen (vs default usage).

**Score → XP mapping:**
- 5/5 = full xpReward from module data
- 4/5 = 80% of xpReward (rounded)
- 3/5 = 60% of xpReward
- 0–2/5 = 40% of xpReward (minimum — never 0)

**No back navigation** on this screen — lesson is committed.

### Screen 5: ReadingScreen

**Layout:**
```
SafeAreaView (midnight bg)
  NavBar: [spacer] ["Reading" label centered] [spacer]
  Reading progress chip (centered): "🎵 About [Artist]" (surfaceHighest, borderRadius 20, caption)
  ScrollView (px md)
    Passage card (surfaceContainer, borderRadius 16, padding lg)
      Passage content — mixed English + tappable Spanish words
      Spanish words rendered as inline pressable chips:
        Resting: coral+22 bg, coral text, borderRadius 4, px 6 py 2
        Pressed: coral bg, white text
    "Continue" CTA (full-width, coral primary)
```

**Tappable word implementation:**
- Passage text split into word tokens at render time
- Spanish words wrapped in `<TouchableOpacity>` inline
- On tap: show Modal or absolute-positioned tooltip card
  - Tooltip: surfaceHigh bg, borderRadius 12, padding md, shadow
  - Content: Spanish word (bodyMedium coral) + English translation (body white) + phonetic if available (caption mist)
  - Dismiss: tap anywhere outside tooltip, or tap word again

**Tooltip position:** Centered on screen (not anchored to word — avoids edge clipping on mobile).

**Ratio display:** No visual ratio indicator in Phase 4 — the passage is pre-authored with the correct A1/A2/B1/B2 ratio baked in.

**No back navigation** — lesson is committed.

## Interaction Contracts

### Timer (ListenScreen)
- Start: tap "I'm listening" → `Animated.timing` counts from `durationSeconds` to 0 over exact duration
- Ring: `Animated.Value` drives `strokeDashoffset` on SVG path — or use `Animated.View` rotation approach with overflow hidden mask
- If song has no `durationSeconds`: default to 210 seconds (3.5 min)
- Auto-complete: when value hits 0, play haptic `notificationAsync('success')`, show "Continue" CTA
- Manual skip: "I'm done listening" → same outcome as auto-complete, no haptic

### Tappable Words (ReadingScreen)
- Passage stored as array of `{ text: string, isSpanish: boolean, english?: string }` tokens
- Render as `<Text>` with inline `<TouchableOpacity>` for Spanish tokens
- Tooltip: absolute positioned `<View>` with `zIndex: 100`, centered, max-width 280px
- Dismiss: `<TouchableWithoutFeedback>` overlay behind tooltip

### Quiz Answer Flow
- Tap option → `setSelectedAnswer(index)` → option shows `selected` state
- Tap "Check Answer" → `setRevealed(true)` → all options show final states, no further changes possible
- `expo-haptics` fires on reveal based on correctness
- "Next Question" → clear state, advance question index, scroll ScrollView to top

### LessonComplete progress update
- `useEffect` on mount: call `progressStore.completeModule(moduleId, xpEarned)`
- `xpEarned` passed as route param from QuizResultsScreen via navigation
- Guard: if `completedModuleIds.includes(moduleId)` → skip (idempotent)

## Navigation Contract

```
ModulesNavigator (Stack)
  ModulesList
  ModuleDetail → "Start Lesson" → LessonNavigator

LessonNavigator (Stack, headerShown: false)
  PreListen
  Listen
  Quiz
  QuizResults
  Reading
  LessonComplete → pop to root (MainTabs)
```

**Entry:** `ModuleDetailScreen` "Start Lesson" CTA navigates to `LessonNavigator` with `{ moduleId }` param.

**Exit points:**
- Abandon (confirm dialog) → `navigation.navigate('MainTabs')` — resets lesson stack
- LessonComplete "Continue to Modules" → `navigation.navigate('MainTabs', { screen: 'Modules' })`
- LessonComplete "Go to Home" → `navigation.navigate('MainTabs', { screen: 'Home' })`

**Back gesture:** Disabled on ListenScreen, QuizScreen, QuizResultsScreen, ReadingScreen, LessonCompleteScreen via `gestureEnabled: false` in navigator options.

**XP passing:** `QuizResultsScreen` receives `{ moduleId, score, total }` and computes `xpEarned`, passes to `ReadingScreen` and then `LessonCompleteScreen` as nav params.
